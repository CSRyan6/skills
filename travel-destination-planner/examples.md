# 示例

## 首轮推荐

用户：

```text
我从厦门出发，周末两日游，想要 3 个候选，火车优先，杭州和长沙去过了。
```

执行方式：

1. 提取输入：
   - 出发地：厦门
   - 出行时长：周末两日
   - 候选数量：3
   - 交通偏好：火车优先
   - `ban_pool`: 杭州、长沙
2. 用 `厦门出发周末两日游` 等关键词搜索小红书。
3. 如果小红书不可用或信息不足，用携程攻略补充。
4. 检查天气和交通。
5. 排除 `ban_pool`。

输出形态：

```markdown
## 候选目的地

### 1. 武夷山
- 推荐理由：小红书笔记中出现“厦门出发”“周末两日”“不爬山悠闲游”等线索，适合两日短途。
- 天气判断：以 nmc.cn / weather.com.cn 查询结果为准；晴或多云优先。
- 交通判断：展示厦门到武夷山方向的 G/D 车次或标注未确认。
- 攻略依据：[小红书笔记标题](https://www.xiaohongshu.com/explore/...)
- 风险/不确定项：如果未拉到具体车次，交通需二次确认。

当前 ban 池：杭州、长沙
```

## 更新 Ban 池

用户：

```text
武夷山也去过了，不想再去。
```

执行方式：

1. 将 `武夷山` 加入 `ban_pool`。
2. 重新生成候选。
3. 不再包含杭州、长沙、武夷山。

输出中应包含：

```markdown
当前 ban 池：杭州、长沙、武夷山
```

## 小红书 461

如果小红书返回 HTTP 461：

```markdown
小红书触发了人机验证，当前不继续高频重试。请先打开 https://www.xiaohongshu.com/explore 完成人工验证；完成后我会低频重试刚才的只读搜索。
```

如果用户希望立即得到候选，再继续使用携程攻略作为补充来源。
